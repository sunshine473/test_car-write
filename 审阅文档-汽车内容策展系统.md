### 二、数据流审阅

#### 模块1：发现热点
- [ ] **信息源完整**：10个懂车帝创作者 + 5个行业Builder + 2个新车网页 + 4个Tavily查询
- [ ] **时间过滤正确**：只保留24小时内的内容
- [ ] **关键词提取**：每条内容必须有3-10个关键词

**审阅问题**：
1. 信息源是否足够？是否需要增加其他平台（如抖音、B站）？
2. 24小时时间窗口是否合适？
3. 懂车帝创作者的10个账号是否需要补充？

#### 模块2：去重与聚类
- [ ] **去重规则明确**：标题相似度>80% 或关键词重叠>60%
- [ ] **聚类规则明确**：每组2-5篇文章，最多8-10个话题组
- [ ] **话题命名清晰**：5-10字主题名

**审阅问题**：
1. 去重阈值（80%、60%）是否合理？
2. 每组2-5篇文章是否足够？
3. 8-10个话题组是否太多或太少？

#### 模块3：筛选与排序
- [ ] **双模型评分**：Gemini初评 + Claude复评 + Claude最终决策
- [ ] **评分维度明确**：热度、新鲜度、市场影响力、写作价值
- [ ] **Top 3筛选**：综合得分≥60分，文章数≥2

**审阅问题**：
1. 双模型评分是否必要？是否可以只用Claude？
2. 4个评分维度是否合理？是否需要调整权重？
3. Top 3是否足够？是否需要Top 5？

#### 模块4：写文章
- [ ] **Tavily深度搜索**：补充至≥20篇素材
- [ ] **双模型写作**：Gemini初稿 + Claude润色
- [ ] **字数控制**：800-1000字（严格）
- [ ] **多平台版本**：5种格式

**审阅问题**：
1. 20篇素材是否足够？
2. 双模型写作是否必要？
3. 5种平台格式是否都需要？

#### 模块5：Telegram发送
- [ ] **交互式推送**：摘要 + 按钮 + 完整文章
- [ ] **用户反馈**：有用/无用
- [ ] **文件下载**：支持PDF/Markdown

**审阅问题**：
1. Telegram推送格式是否满意？
2. 是否需要其他推送方式（邮件、微信）？
3. 用户反馈是否需要更详细的选项？

### 六、时间与成本审阅

#### 预估时间
- **模块1（发现热点）**：5-10分钟（取决于网络速度）
- **模块2（去重聚类）**：2-3分钟（Claude API调用）
- **模块3（筛选排序）**：3-5分钟（双模型评分）
- **模块4（写文章）**：10-15分钟（Tavily搜索 + 双模型写作）
- **模块5（Telegram发送）**：1分钟

**总计**：约20-35分钟/天

**审阅问题**：
1. 时间预估是否合理？
2. 是否可以接受每天30分钟的运行时间？
3. 是否需要优化性能？

#### 预估成本
- **Tavily API**：约$0.01-0.02/天（25-30次查询）
- **Gemini API**：约$0.05-0.10/天（评分 + 写作）
- **Claude API**：约$0.10-0.20/天（评分 + 润色）
- **Telegram Bot**：免费

**总计**：约$0.16-0.32/天，$5-10/月

**审阅问题**：
1. 成本是否可以接受？
2. 是否需要降低API调用次数？
3. 是否可以用更便宜的模型？

### 七、风险与应对

#### 潜在风险
1. **反爬风险**：懂车帝可能封IP
   - **应对**：使用代理、降低请求频率、使用CDP浏览器
2. **API限流**：Tavily/Gemini/Claude可能限流
   - **应对**：添加重试机制、错误处理
3. **内容质量**：AI生成的文章可能不符合预期
   - **应对**：人工审核、调整Prompt
4. **时效性**：某些信息源可能更新不及时
   - **应对**：增加信息源、缩短时间窗口

**审阅问题**：
1. 是否有其他潜在风险？
2. 应对措施是否充分？
3. 是否需要增加监控和告警？
