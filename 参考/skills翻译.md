# Skills 技能翻译文档

本文档提供了参考目录中所有技能的中文翻译和说明。

## 1. doc-coauthoring (文档协作撰写)

**用途**: 引导用户通过结构化工作流程协作撰写文档。当用户想要编写文档、提案、技术规范、决策文档或类似的结构化内容时使用。

**工作流程**:
- **阶段1：上下文收集** - 用户提供所有相关上下文，Claude提出澄清问题
- **阶段2：细化与结构** - 通过头脑风暴和编辑迭代构建每个部分
- **阶段3：读者测试** - 用全新的Claude实例（无上下文）测试文档，以捕捉盲点

**触发条件**: 用户提到编写文档、创建提案、起草规范或类似的文档任务。

## 2. canvas-design (画布设计)

**用途**: 使用设计哲学在.png和.pdf文档中创建精美的视觉艺术。当用户要求创建海报、艺术作品、设计或其他静态作品时使用。

**工作流程**:
1. **设计哲学创建** - 创建一个视觉哲学（美学运动），通过形式、空间、色彩、构图来表达
2. **画布表达** - 在画布上创建作品，体现设计哲学

**核心原则**:
- 90%视觉设计，10%必要文本
- 强调专家级工艺，看起来像花费了无数小时精心制作
- 避免重复，每个设计方面只提及一次
- 使用Anthropic品牌（UI颜色、字体、布局）

**输出**: .md文件（设计哲学）+ .pdf或.png文件（最终作品）

## 3. claude-api (Claude API开发)

**用途**: 使用Claude API或Anthropic SDK构建应用程序。当代码导入`anthropic`/`@anthropic-ai/sdk`/`claude_agent_sdk`，或用户要求使用Claude API时触发。

**默认设置**:
- 模型: Claude Opus 4.6 (`claude-opus-4-6`)
- 思考模式: 自适应思考 (`thinking: {type: "adaptive"}`)
- 对于长输入/输出使用流式传输

**选择合适的层级**:
- **单次LLM调用**: 分类、摘要、提取、问答 → 使用Claude API
- **工作流**: 多步骤管道，代码控制逻辑 → Claude API + 工具使用
- **代理**: 需要文件/网络/终端访问 → Agent SDK

**语言支持**: Python, TypeScript, Java, Go, Ruby, C#, PHP, cURL

**品牌颜色**:

## 5. algorithmic-art (算法艺术)

**用途**: 使用p5.js创建具有种子随机性和交互式参数探索的算法艺术。当用户请求使用代码创建艺术、生成艺术、算法艺术、流场或粒子系统时使用。

**工作流程**:
1. **算法哲学创建** - 创建计算美学运动
2. **p5.js实现** - 通过代码表达哲学

**技术要求**:
- 种子随机性（Art Blocks模式）
- 参数结构（种子、颜色、数量、比例等）
- 画布设置（标准p5.js结构）

**输出**: 单个自包含的HTML工件（包含p5.js、算法、参数控制和UI）

**关键特性**:
- 种子导航（上一个/下一个/随机按钮）
- 参数控制（滑块、颜色选择器）
- 实时更新

## 7. pdf (PDF处理)

**用途**: 处理PDF文件的任何操作。包括读取或提取文本/表格、合并或拆分PDF、旋转页面、添加水印、创建新PDF、填写PDF表单、加密/解密PDF、提取图像、OCR扫描PDF。

**主要库**:
- **pypdf**: 基本操作（合并、拆分、旋转、元数据）
- **pdfplumber**: 文本和表格提取
- **reportlab**: 创建PDF

**常见任务**:
- 合并PDF: `pypdf`
- 提取文本: `pdfplumber`
- 提取表格: `pdfplumber`
- 创建PDF: `reportlab`
- OCR扫描PDF: `pytesseract` + `pdf2image`

**命令行工具**: pdftotext, qpdf, pdftk

## 8. mcp-builder (MCP服务器构建)

**用途**: 创建高质量的MCP（模型上下文协议）服务器，使LLM能够通过精心设计的工具与外部服务交互。

**推荐技术栈**:
- **语言**: TypeScript（推荐）
- **传输**: 远程服务器使用可流式HTTP，本地服务器使用stdio

**四个主要阶段**:
1. **深入研究和规划** - 理解现代MCP设计、研究API文档
2. **实现** - 设置项目结构、实现核心基础设施和工具
3. **审查和测试** - 代码质量审查、构建和测试
4. **创建评估** - 创建10个复杂的评估问题来测试有效性

## 9. frontend-design (前端设计)

**用途**: 创建独特的、生产级的前端界面，具有高设计质量。当用户要求构建网页组件、页面、工件、海报或应用程序时使用。

**设计思维**:
- **目的**: 这个界面解决什么问题？谁使用它？
- **基调**: 选择一个极端方向（极简、极繁、复古未来、有机自然等）
- **差异化**: 什么让这个令人难忘？

**关键要素**:
- **排版**: 选择美丽、独特、有趣的字体
- **颜色与主题**: 承诺一个连贯的美学
- **动效**: 使用动画效果和微交互
- **空间构图**: 意外的布局、不对称、重叠

**避免**: 通用AI生成的美学（Inter字体、紫色渐变、可预测的布局）

## 10. docx (Word文档)

**用途**: 创建、读取、编辑或操作Word文档（.docx文件）。包括提取内容、插入/替换图像、查找替换、处理跟踪更改或注释、转换内容为精美的Word文档。

**快速参考**:
- **读取/分析内容**: `pandoc`或解包查看原始XML
- **创建新文档**: 使用`docx-js`
- **编辑现有文档**: 解包 → 编辑XML → 重新打包

**创建新文档**:
- 使用JavaScript和docx-js库
- 设置页面大小（默认A4，美国信纸需明确设置）
- 样式（覆盖内置标题）
- 列表（使用编号配置，不要使用unicode项目符号）
- 表格（需要双重宽度设置）
- 图像、分页符、超链接、脚注等

**编辑现有文档**:
1. 解包: `python scripts/office/unpack.py document.docx unpacked/`
2. 编辑XML
3. 打包: `python scripts/office/pack.py unpacked/ output.docx --original document.docx`

## 11. pptx (PowerPoint演示文稿)

**用途**: 涉及.pptx文件的任何操作。包括创建幻灯片、读取/解析/提取文本、编辑/修改现有演示文稿、合并或拆分幻灯片文件。

**快速参考**:
- **读取/分析内容**: `python -m markitdown presentation.pptx`
- **编辑或从模板创建**: 阅读editing.md
- **从头创建**: 阅读pptxgenjs.md

**设计理念**:
- 不要创建无聊的幻灯片
- 选择大胆的、内容相关的调色板
- 主导色而非平等（一种颜色占60-70%视觉权重）
- 每张幻灯片都需要视觉元素（图像、图表、图标或形状）

**排版**: 选择有趣的字体配对，不要默认使用Arial

**QA要求**: 假设存在问题，你的工作是找到它们。使用子代理进行视觉QA。

## 12. web-artifacts-builder (Web工件构建器)

**用途**: 使用现代前端网络技术（React、Tailwind CSS、shadcn/ui）创建复杂的、多组件的claude.ai HTML工件。用于需要状态管理、路由或shadcn/ui组件的复杂工件。

**技术栈**: React 18 + TypeScript + Vite + Parcel（打包）+ Tailwind CSS + shadcn/ui

**工作流程**:
1. 初始化前端仓库: `bash scripts/init-artifact.sh <project-name>`
2. 开发工件（编辑生成的代码）
3. 打包为单个HTML文件: `bash scripts/bundle-artifact.sh`
4. 向用户展示工件
5. （可选）测试工件

**重要**: 避免"AI slop"，不要使用过度居中的布局、紫色渐变、统一的圆角和Inter字体。

## 14. slack-gif-creator (Slack GIF创建器)

**用途**: 为Slack创建优化的动画GIF的知识和实用工具。当用户请求为Slack创建动画GIF时使用。

**Slack要求**:
- **尺寸**: 表情符号GIF 128x128，消息GIF 480x480
- **参数**: FPS 10-30，颜色 48-128，时长<3秒（表情符号）

**核心工作流程**:
1. 创建构建器: `GIFBuilder(width=128, height=128, fps=10)`
2. 生成帧（使用PIL绘制）
3. 保存并优化: `builder.save('output.gif', num_colors=48, optimize_for_emoji=True)`

**动画概念**: 抖动/振动、脉冲/心跳、弹跳、旋转、淡入/淡出、滑动、缩放、爆炸/粒子爆发

## 16. xlsx (Excel电子表格)

**用途**: 当电子表格文件是主要输入或输出时使用。包括打开、读取、编辑或修复现有的.xlsx、.xlsm、.csv或.tsv文件；从头创建新电子表格；在表格文件格式之间转换。

**关键要求**:
- **零公式错误**: 每个Excel模型必须以零公式错误交付
- **专业字体**: 使用一致的专业字体（如Arial、Times New Roman）
- **使用公式，不要硬编码值**: 始终使用Excel公式而不是在Python中计算值并硬编码

**主要库**:
- **pandas**: 数据分析、批量操作、简单数据导出
- **openpyxl**: 复杂格式化、公式、Excel特定功能

**工作流程**:
1. 选择工具（pandas用于数据，openpyxl用于公式/格式化）
2. 创建/加载工作簿
3. 修改（添加/编辑数据、公式和格式）
4. 保存文件
5. **重新计算公式（如果使用公式则必须）**: `python scripts/recalc.py output.xlsx`
6. 验证并修复任何错误

**财务模型颜色编码**:
- 蓝色文本：硬编码输入
- 黑色文本：所有公式和计算
- 绿色文本：同一工作簿内其他工作表的链接
- 红色文本：其他文件的外部链接
- 黄色背景：需要注意的关键假设

## 17. webapp-testing (Web应用测试)

**用途**: 使用Playwright与本地Web应用程序交互和测试的工具包。支持验证前端功能、调试UI行为、捕获浏览器截图和查看浏览器日志。

**决策树**:
1. 是静态HTML吗？
   - 是 → 直接读取HTML文件识别选择器
   - 否 → 服务器已运行吗？
     - 否 → 使用`scripts/with_server.py`
     - 是 → 侦察然后行动模式

**侦察然后行动模式**:
1. 检查渲染的DOM（截图、获取内容）
2. 从检查结果中识别选择器
3. 使用发现的选择器执行操作

**最佳实践**:
- 使用`sync_playwright()`进行同步脚本
- 始终在完成后关闭浏览器
- 使用描述性选择器
- 添加适当的等待：`page.wait_for_selector()`或`page.wait_for_timeout()`

**关键陷阱**: 在动态应用上，不要在等待`networkidle`之前检查DOM

## 总结

这些技能涵盖了广泛的任务类型：

- **文档处理**: doc-coauthoring, docx, pdf, pptx, xlsx
- **设计与艺术**: canvas-design, algorithmic-art, frontend-design, theme-factory
- **开发工具**: claude-api, mcp-builder, web-artifacts-builder, webapp-testing
- **沟通与品牌**: internal-comms, brand-guidelines, slack-gif-creator
- **元工具**: skill-creator

每个技能都有明确的触发条件、工作流程和最佳实践，旨在帮助Claude更有效地完成特定类型的任务。
