## 🎓 经验总结

### 成功经验
1. **架构设计先行** - 先设计再实现
2. **文档驱动开发** - 详细文档帮助理清思路
3. **测试驱动调试** - 运行测试发现真实问题
4. **快速迭代修复** - 发现问题立即修复

### 遇到的挑战
1. **API 不稳定** - Gemini 经常 503
2. **JSON 解析困难** - Claude 返回格式不规范
3. **数据去重过严** - 导致可用数据太少
4. **并行执行复杂** - 需要仔细设计

### 解决方案
1. **备用方案** - Gemini 失败用 Claude
2. **改进 Prompt** - 明确要求返回格式
3. **调整策略** - 放宽去重标准
4. **独立 Agent** - 每个 Writer 独立运行
