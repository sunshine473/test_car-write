# 汽车内容策展系统 · 完整需求与实现文档

## 📋 一、项目背景与目标

### 问题
你现在有一个「懂车帝车型采集工具」，能抓取某一款车的详细参数。但你需要的是一个**更高层的系统**：
1. **发现热点** — 每天自动搜索汽车行业的最新新闻、讨论、趋势
2. **确认选题** — 从热点中筛选出最值得写的文章主题
3. **生成内容** — 自动写出完整文章

### 目标
构建一个**汽车内容策展系统（car-content-curator）**，类似于你参考的「follow-builders」项目：
- follow-builders：每天自动搜集 AI 领域的热点（X 推文、播客、博客），筛选重点，生成日报
- car-content-curator：每天自动搜集汽车行业的热点，筛选重点，**生成完整文章**

**最终形态**：用户设置一次后，系统每天自动找选题、写好文章、通过 Telegram 推送。

## 📊 三、模块详细设计

### 模块1：发现热点

#### 输入
1. **配置文件**：`config/car-sources.json`
2. **时间参数**：当前时间，时间窗口24小时
3. **API密钥**：从 `.env` 读取

#### 处理流程
1. 读取配置文件，获取所有 `enabled: true` 的信息源
2. 按信息源类型执行抓取：
   - 懂车帝创作者：访问用户主页，获取当天视频/文章
   - 行业Builder微博：抓取当天微博发言
   - 新车发布网页：爬取新车频道
   - Tavily搜索：执行4个查询，每个获取5条结果
3. 时间过滤：只保留24小时内发布的内容
4. 提取关键词：从标题+摘要中提取3-10个关键词

### 模块2：去重与聚类

#### 输入
**文件**：`data/feeds/feed-2026-04-13.json`（42条热点）

#### 处理流程
1. **去重**：
   - 标题完全相同 → 保留热度最高的
   - 标题相似度 > 80% → 合并
   - 关键词重叠 > 60% 且时间相近 → 判定为同一事件
2. **聚类**：
   - 基于关键词重叠度分组
   - 每组2-5篇文章
   - 最多输出8-10个话题组
3. **话题命名**：
   - 提取共同关键词
   - 生成5-10字主题名

### 模块3：自动筛选与排序

#### 输入
**文件**：`data/clustered/clustered-2026-04-13.json`（8个话题组）

#### 处理流程
1. **Gemini 初评**：
   - 读取 `config/ranking-prompts.json` 中的 `gemini_prompt`

2. **Claude 复评**：
   - 读取 `claude_prompt`
   - 参考 Gemini 结果，独立评分
3. **Claude 最终决策**：
   - 读取 `claude_final_decision_prompt`
   - 综合两个模型评分
   - 选出 Top 3 话题

### 模块4：写文章

**文件**：`data/ranked/ranked-2026-04-13.json`（Top 3话题）

1. **Tavily 深度搜索**：
   - 基于话题关键词构造查询
   - 至少获取20篇相关文章
   - 合并模块3的相关文章
2. **素材准备**：
   - 访问所有链接（WebFetch 或 CDP）
   - 提取完整内容
3. **Gemini 生成初稿**：
   - 读取 `config/writing-prompts.json` 中的 `gemini_draft_prompt`
   - 生成800-1000字文章
4. **Claude 优化润色**：
   - 读取 `claude_polish_prompt`
   - 优化可读性、吸引力
5. **多平台版本生成**：
   - 读取 `platform_prompts` 中各平台的 Prompt
   - 生成知乎、小红书、YouTube、今日头条、微信版本

### 模块5：Telegram发送

#### 输入
**文件**：`data/articles/article-topic_001-2026-04-13.json`

#### 处理流程
1. **消息格式化**：
   - 读取 `config/telegram-config.json` 中的 `message_templates`
   - 生成摘要消息
   - 创建交互按钮
2. **发送摘要**：
   - 使用 Telegram Bot API
   - 发送 Top 3 话题摘要
3. **处理用户交互**：
   - 用户点击按钮 → 发送对应平台的完整文章
   - 用户点击下载 → 发送 PDF 文件

   - 收集"有用/无用"反馈

## 📋 五、总结

这个系统的核心思路：

```
大量热点信息 → 智能去重聚类 → 双模型评分 → 自动写文章 → Telegram推送
```

**核心价值**：为你省去手动搜索、分析、写作的时间，让你专注于内容审核和发布。

**技术栈**：
- 数据采集：Tavily API、Web爬虫、CDP
- AI模型：Gemini 2.5 Pro、Claude Opus 4.6
- 推送：Telegram Bot API
- 存储：JSON文件
- 配置：所有Prompt和参数都在配置文件中，不写入硬编码
